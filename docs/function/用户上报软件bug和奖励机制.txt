1.增加报告系统bug或改进能力，使用户极其方便的上报，并给予奖励（贡献值等虚拟奖励）；
2.让软件成为社区自行维护的一个圈子，增加用户黏性和参与感；
3.用户贡献值可以放入目前用户评分体系内，后期可以拓展到即将打造的Plaza内去完善和进一步利用。  

用户档案上显示两个独立标签:

💼 交易信用 ⭐4.8(基于成交评价)
🎖️ 社区贡献 Lv.3(基于反馈、未来 Plaza 互动)

合并展示但分开计算。这样:

买家看交易信用决定要不要买
大家看社区贡献感知用户的"参与度"
两者不互相污染语义

关键决策:贡献值用流水账而不是单纯一个总数字段。理由:

可追溯("我这 50 分都是哪来的?")
可审计(admin 误操作可以撤销)
未来扩展灵活(Plaza 的贡献来源也只是新增流水类型)

App 端

摇一摇监听 + 截图 + 涂鸦组件(Flutter 有现成包 feedback package)
"我的贡献"页面:总分 + 流水 + 徽章 + 我提交过的反馈状态
Push 通知钩子(反馈状态变化时推送)

Admin 端

反馈处理队列(类似举报队列,但走"产品视角"而不是"风控视角")
一键发放贡献值的工作流(选状态 → 自动加分 → 发推送)
用户贡献值排行榜(可选,用作内部运营参考)

Edge Function

提交反馈(带防滥用:每用户每天最多 5 条)
Admin 处理反馈时,事务性写入流水 + 更新徽章 + 发推送

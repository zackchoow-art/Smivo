# 校园共享拼车（Carpool）功能开发计划书

## 1. 基础设施与系统设置 (Foundation & Admin)
### 1.1 系统服务配置
- **API Key 存储**：在 `system_configurations` 表中增加 `google_maps_places_api_key` 和 `google_maps_directions_api_key`。
- **管理后台 (React)**：在 Admin Dashboard 增加“地图服务配置”页面，供管理员维护全局 API Key。
- **学校差异化设置**：在“学校管理”页面增加 `ios_default_map_provider` 字段（选项：Apple Maps / Google Maps），决定该校 iOS 用户的默认地图服务。

### 1.2 成本优化 (Cost Control)
- **Session Token 机制**：针对 Google Places Autocomplete 实现 `sessionToken` 逻辑。每次地址搜索会话生成一个 UUID，并在该会话的所有请求中携带，使 Google 将多次输入计为单次计费会话。

## 2. 地图与发布逻辑 (Mapping & Initiation)
### 2.1 动态地图引擎
- **引擎集成**：同时集成 `apple_maps_flutter` 和 `google_maps_flutter`。
- **分发策略**：
    - **iOS**：根据后台学校配置动态调用 Apple Maps 或 Google Maps。
    - **Android/Web**：强制调用 Google Maps。
- **地理位置权限**：集成 `geolocator`，申请 GPS 定位权限。

### 2.2 精准选址与预览
- **交互式选址**：通过地图拾取器获取精确经纬度及标准化地址（Place ID）。
- **路线与 ETA**：调用 `Google Directions API` 在发布页绘制路线预览并呈现预估到达时间（ETA）。

### 2.3 拼车告示发布
- **多样化角色**：支持“司机提供车辆”和“发起人找人分摊打车费”两种模式。
- **发布项**：出发时间、集合地点、目的地、座位数（1-4）、行李限额、审核模式（自动/手动）、拼车关闭时间。

## 3. 安全校验与加入逻辑 (Safety & Joining)
### 3.1 社交屏蔽联动
- **黑名单预检**：加入申请时，自动校验申请人与群内“所有”现有成员之间是否存在 `user_blocks` 记录。
- **智能拦截**：若存在拉黑关系，给出礼貌提醒并拦截申请，确保拼车环境友好。

### 3.2 容量自动化管理
- **人满下架**：加入人数达到上限时，数据库触发器自动将 Listing 设为 `inactive`。
- **空位补录**：有人退出导致座位空余时，立即自动恢复 Listing 为 `active`。

## 4. 协作协作与共识流 (Collaboration & Consensus)
### 4.1 自动化群聊
- **即时入群**：成员被接受后，系统自动将其关联至该行程的专属群聊（Group Chat）。
- **实时同步**：基于 Supabase Realtime 确保行程讨论的即时性。

### 4.2 提议与投票机制
- **行程变更共识**：发起人修改时间/地点后，需获得行程内“所有”成员的点击确认，修改方可生效。
- **投票踢人逻辑**：发起人可针对无响应或异常成员发起投票，通过后自动移除成员并重新释放座位。

## 5. 行程闭环与社交功能 (Post-Trip & Social)
### 5.1 到达确认
- **状态检测**：以 `ETA + 缓冲时间` 为判定点。
- **唤醒引导**：用户在行程后首次打开 App 时，引导确认：“您已到达目的地吗？”。

### 5.2 全员网状评价 (N-to-N Reviews)
- **多方互评**：行程结束后，提供批量评价界面。
- **评分闭环**：每位参与者必须对行程中的其他所有人进行独立评分，构建拼车信用体系。

### 5.3 系统日历深度集成
- **自动化同步**：在用户确认的前提下，自动将行程信息（含时间、地点、提醒时间）写入手机原生日历。
- **更新维护**：支持在行程变更后同步更新已有的日历条目。

## 6. 风险提示与合规
- **法律声明**：显著位置展示“校园互助免责声明”，严禁非法营运和营利。
- **架构集成**：作为 Smivo App 核心功能直接集成（非独立 Package），确保用户体验的一致性。

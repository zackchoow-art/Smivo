二、对现有页面的改造
1、合并目前的Dashboard和Data Dashboard页面，整合成一个统一的统计和数据分析页面，

2、左侧边栏的User reports改名为Listing reports。

3、Data dictionary页面中的System 模块和 Platform 模块大多数都会搬走，这里只剩下了School模块的三个设置，所以这个页面改名为School Settings，供school reviewer查看，school admin 可维护修改。

4、Platform settings页面中的Feature Flags标签页里的所有内容都在新建的Platform Functions页面里了，所以可以删除这个标签页，然后给Platform settings改名为Content Moderation。在这个页面的App-side settings部分增加一个content_filter.warn_message，在Server-side settings部分增加image_moderation_mode选项。

5、system configuration页面中的所有内容都或因重复而删除，或因搬到新页面而删除，所以这个页面基本可以删除。



三、功能改进的说明
对于某些硬编码尚未实现平台管理维护的部分，Gemini flash做了一份调查报告和建议，请参考system_configuration_audit_report.md文件，你可以根据你的判断决定是否采纳。

对所有的功能设置，添加详细的说明，说明这些设置的作用和影响，以及对应数据库的哪个表哪个字段（或者其他形式的存储）。

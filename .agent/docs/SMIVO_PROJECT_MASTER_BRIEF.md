# SMIVO: 核心项目全景简报 (Master Project Brief)

> **文档定位**：本文件作为 Smivo 项目的最高级别架构与产品说明书，专供新接手的 Agent 或开发者快速理解项目全貌、业务逻辑及技术规范。

## 1. 软件介绍与愿景 (Introduction & Vision)
**Smivo** 是一款专为高校学生打造的 C2C（校园二手与租赁）高信任度交易平台与校园社区。
应用首发站为 **Smith College**。产品的核心愿景是通过 `.edu` 邮箱的严格身份壁垒，建立一个只有同校真实校友才能参与的“安全交易与交流圈”，并计划未来通过“一校一区”的模式向全美其他高校横向扩张。

## 2. 致力于解决的核心问题 (Problem Solved)
- **信任与安全短板**：解决 Facebook Marketplace 和 Craigslist 等平台因身份不明导致的面交安全隐患和诈骗问题。
- **租赁摩擦成本**：传统闲鱼模式只支持“买卖”。Smivo 原生支持“时间计费租赁（日/周/月）”，并自带押金流转机制，满足高校生对于课本、礼服、电子产品等短期急用的租赁痛点。
- **沟通效率**：将零散的讨价还价整合为结构化的“交易大厅（Transaction Management）”与订单全生命周期追踪。

## 3. 技术架构与技术路线 (Architecture & Tech Stack)

Smivo 采用 **客户端分离 + Serverless 中台** 的现代化 Monorepo 架构。

### 3.1 前端应用栈 (Frontend)
- **Mobile App (iOS/Android/Web)**：使用 **Flutter** 构建。
  - **状态管理**：`Riverpod` (Code generation, AsyncNotifier)。
  - **路由控制**：`GoRouter` (支持 Universal Links 深度链接跳转)。
  - **UI 与设计语言**：使用 Google Stitch 绘制界面。自研双主题系统（Smivo Teal 圆角玻璃拟物风 & IKEA 高对比度网格粗野风）。
- **Admin Control Center (管理后台)**：使用 **React.js + Vite + Tailwind CSS** 构建（独立存放在 `admin_web` 目录）。
- **官网落地页**：纯静态 HTML/CSS（存放在 `website` 目录），托管于 Vercel。

### 3.2 后端与基础设施层 (Backend - Supabase)
完全依托 **Supabase (BaaS)** 实现：
- **数据库**：PostgreSQL (启用 Row Level Security - RLS 行级权限控制)。
- **身份验证**：Supabase Auth (仅放行特定 `.edu` 邮箱后缀注册)。
- **实时通信**：Supabase Realtime (用于实时聊天消息推送)。
- **云端存储**：Supabase Storage (图片、订单交接凭证等)。
- **无服务器函数**：Edge Functions (处理 OneSignal 推送、敏感词过滤等特权逻辑)。

## 4. 核心数据结构与关系模型 (Data Structure)

数据库严格遵循强类型外键关联，主要实体包括：
- `user_profiles`: 用户核心档案（包含 `last_active_at`，支持心跳在线追踪）。
- `listings` & `listing_images`: 商品与图片表（包含售卖与租赁两种计费模式）。
- `orders`: 订单流水表（极其核心，涵盖 pending/active/completed 生命周期）。
- `order_evidence`: 交易凭证库（存放当面交付与归还时的拍照留存记录）。
- `chat_rooms` & `messages`: 聊天室与消息表（基于清单 ID 绑定买卖双方）。
- `rental_extensions`: 租赁延期/缩短协商表。
- **(即将接入)** `hourly_active_users`: 在线时段统计时间桶。
- **(即将接入)** `user_feedbacks` & `forum_posts`: 校园广场与意见反馈表。

## 5. 软件功能与交互逻辑 (Features & Interaction Flow)

### 5.1 已经实现的核心功能 (Implemented)
1. **严格的身份鉴权**：游客可自由浏览，但发起交易或聊天必须使用 `.edu` 邮箱注册。
2. **商品发布与双模交易**：支持一口价出售或按周期出租。
3. **Transaction Dashboard (卖家交易控制台)**：卖家可在一个大屏内管理对同一件商品的所有报价。接受某个买家的 offer 后，通过数据库 RPC 原子化自动拒绝其他排队买家。
4. **租赁订单状态机**：支持“双方共同确认收货”、“租赁延期谈判”、“发起归还请求”和“退还押金”的全链路管控。
5. **内嵌实时聊天 (Realtime Chat)**：与特定订单绑定的私聊窗口，无需刷新即可实时接收消息。
6. **全链路推送通知**：接入 OneSignal 配合 Supabase Edge Function，实现业务动作触发系统级推送。
7. **全平台 Universal Links**：分享商品可生成 `smivo.io/listing/xxx` 深度链接，App 内直接唤醒，Web 端降级跳转主页。

### 5.2 计划实施的功能池 (Planned Roadmap)
1. **React Admin Web (最高优先级)**：从零构建基于 React 的高颜值管理大盘，接管风控审核、推送下发、DAU统计与全局配置。
2. **UGC 智能审核防火墙**：集成 `bad-words` 敏感词词库，发帖发信前自动过滤/拦截违规词汇。
3. **Campus Plaza (校园广场/树洞)**：新增 UGC 社区模块，支持纯文本分享、吐槽、失物招领，打造强粘性高频应用场景。
4. **意见与反馈系统**：建立 App 端向 Admin 端直连的 Bug/需求上报通道。
5. **心跳状态基建 (Presence System)**：基于本地防抖与时间桶技术，极低成本实现全端“绿点”在线状态指示。

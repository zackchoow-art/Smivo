# Smivo 项目修改需求清单 (Master Requirements List)

版本：v3.0
日期：2026-04-23

---

## 1. 卖家统计与“交易管理”页面 (Listing Stats & Transaction Management)
*   **入口按钮**：在商品详情页的卖家专属统计区域（显示 Views/Saves/Orders 的区域）下方，新增一个按钮**“交易管理 (Manage Transactions)”**。
*   **独立页面实现**：点击上述按钮后，导航至一个新的独立页面，该页面承载原本的“用户穿透列表”功能。
*   **页面核心功能**：
    *   **分区统计与列表穿透**：
        *   **查看 (Views)**：显示列表（用户名称/评分/过往卖出买入数量/查看次数/私信按钮）。*注：需排除卖家本人。*
        *   **收藏 (Saves)**：显示列表（用户名称/评分/过往卖出买入数量/收藏时间/私信按钮）。
        *   **下单 (Orders)**：显示列表（用户名称/评分/过往卖出买入数量/下单时间/私信按钮）。
    *   **核心操作**：在“下单”列表中，针对每个待处理申请提供**“同意交易 (Accept)”**按钮。

## 2. 商品详情页 - 状态逻辑与 UI 优化 (Listing Detail Screen)
*   **买家下单状态**：若当前买家已提交申请，按钮替换为“已提交申请”及具体提交时间。
*   **收藏功能**：激活 Save 功能；发布者本人隐藏此按钮且不可操作。
*   **UI 修正**：
    *   固定返回按钮位置，不随滚动消失。
    *   图片标签改为显示成色（如“New / Like New”）。
    *   “about this item”改回“Description”并移至名称下方。
    *   移除租赁商品下方的“Smith College Campus”冗余文本。
*   **布局修复**：修复 Rental Period 与 Total 金额行的溢出问题，采用两行布局并垂直居中金额。

## 3. 卖家中心 (Seller Center)
*   **架构定义**：将原“My Orders”的卖家部分独立为新页面。
*   **上架商品分区**：显示所有在线商品；成交后自动下架；点击进入商品详情页。
*   **已成交商品分区**：展示历史卖出/租出记录及订单状态。

## 4. 买家中心 (Buyer Center)
*   **架构定义**：将原“My Orders”的买家部分独立为新页面。
*   **求购区 (Requested)**：显示待处理申请；卖家接受则进入“已买到”，被拒或卖家选择他人则标记为“Missed”；点击进入详情页（按钮锁定，不可重复申请）。
*   **已买到商品区 (Purchased)**：展示历史买入/租入记录。

## 5. 租赁订单逻辑修复 (Rental Order Logic)
*   **状态流转**：双方确认收货后进入“Active”区域，而非直接进入“History”。
*   **生命周期**：新增“归还商品”及“退还押金”流程，完成后才转入“History”。

## 6. 订单详情、快照与安全性 (Orders & Security)
*   **租赁详情**：增加显示成交时的租赁方式、单价、总价。
*   **拍照取证功能**：在确认收货前支持上传最多 5 张取证照片，作为证据留存。
*   **详细快照页**：升级为独立详情页面（而非抽屉），集成：
    *   全流程时间轴（包含租赁状态变更的所有时间戳）。
    *   财务汇总（成交金额、押金、单价及租赁方式）。
    *   位置信息 (Pickup Location)。
    *   照片证据。
    *   **双方完整聊天记录**。
*   **取消锁定**：在确认收货节点后锁定买卖双方的取消/撤回权限，消除隐患。

## 7. 产品发布页 (Product Posting Page)
*   **视觉简化**：移除顶部页面标题及假消息图标。
*   **功能裁剪**：移除置顶复选框及费用计算，延后竞价与付费功能。
